Using Color Overrides

To customize Material button and FAB colors with your theme’s CSS variables, import the provided SCSS overrides in your global or feature styles:

This will asllow you apply your theme’s primary, secondary, tertiary, and error colors to all Material buttons, FABs and other Mat Components using CSS variables. You can see this in action in our Theme Showcase demo. 

@use '@spider-baby/material-theming/styles/mat-color-overrides.scss' as *;

Button Color Overrides: Example

Below you can see the code to use a themed button, and the actual result: 

<button mat-flat-button class="primary">Primary</button>
<button mat-flat-button class="secondary">Secondary</button>
<button mat-flat-button class="tertiary">Tertiary</button>

API Documentation

For detailed information about all available methods and properties, see the article ThemeService API Documentation

The ThemeService provides methods for applying themes, switching between light and dark mode, creating custom themes, and more. Explore the complete API to get the most out of the theming system.

Note: The Theming System will only be initialized if the ThemeService is injected at least once in your app,
or if you are using our MlThemePickerMatComponent or MlDarkModeToggleMatComponent .

If not ensure your theme variables are always generated, add provideThemeInitializer() to your app’s providers.
Example:
bootstrapApplication(AppComponent, { providers: [provideThemeInitializer(THEME_CONFIG)] })
